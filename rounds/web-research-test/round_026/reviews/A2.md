## Possible errors or hidden assumptions

1.  **A3's Numerical Scale Assumption:** A3 executes numerical tests for R5 and M9 at $X=16$ and $X=256$. These scales are too small to capture asymptotic behavior, dyadic overlap losses, or the density of near-collision bands. They function as formula regression checks. Extrapolating asymptotic control from $X=256$ is a hidden assumption that should be rejected.
2.  **R5 Dyadic Overlap Loss (A1):** A1's R5-Full proof states that logarithmic losses from dyadic decomposition are absorbed into $X^\epsilon$. This assumes that the smooth dyadic partition $w_D(d)$ has bounded overlap (e.g., at most 2 non-zero weights for any $d$). If the partition is not constructed carefully, the sum over dyadic blocks could inflate the implicit constant.
3.  **Complex Weight Failure in Paired Formulas (A1, A3):** The paired real formulas for $\mathcal M_1$ and $\mathcal M_2$ rely on $A_{-h}(D;X) = \overline{A_h(D;X)}$, which requires $w_D(d)$ to be real. If complex weights are introduced (e.g., during a spectral decomposition or Mellin transform), the paired formulas no longer apply. This assumption should be explicitly flagged in numerical implementations.
4.  **H13 Operator-Norm Collapse (A3):** A3's finite H13 endpoint test concludes that the dual character $\chi_4(m)$ appears as diagonal unitary conjugation, making it character-blind under operator norm. This assumes that the first post-transform step is a standard Cauchy-Schwarz inequality leading to an operator norm. If a direct signed bilinear form or a non-conjugacy trace is used instead, the character might survive. The diagnostic is bounded to specific norm extractions.

## Additional theorem-hypothesis audit

The current A1/A3 package should be promoted only where the hypotheses are visible at the formula level. For H4, the review should record the distinction between the centered periodic sawtooth and the arithmetic floor-compatible sawtooth. The exact conversion at integers is
$$
\psi_F(n)=-\frac12,\qquad V_H(n)=0,\qquad {K_H(0)\over 2H+2}=\frac12.
$$
This is enough for the endpoint convention, but the proof draft should avoid implying that Vaaler's centered theorem itself has the floor-compatible point value. The source theorem supplies a centered inequality; the arithmetic identity supplies the point convention; the Fejer majorant bridges them.

For R5-Full, the theorem input is not cancellation. It is positivity of the Fejer kernel plus an arithmetic product count. The first residual leg uses
$$
{1\over H}K_H(X/d)
\ll
W_\Delta(X-md),
\qquad
\Delta={D\over H}\asymp X^{1/4},
$$
where $m$ is a nearest integer to $X/d$. The shifted legs use products $X\approx d(4m-\rho)$ with $\rho\in\{1,3\}$. The review should therefore require every future R5 table to list exact resonances separately from near resonances. A table that only prints a floating-point sine quotient can miss the case $\sin \pi t=0$ and distort the diagnostic.

For M9, the theorem target should stay fixed-coefficient. The two main sums contain
$$
\alpha_{h,H_D}
=-\frac{\Phi(|h|/(H_D+1))}{2\pi i h},
\qquad
\Phi(u)=\pi u(1-u)\cot(\pi u)+u.
$$
Replacing these coefficients by arbitrary phases or by $|\alpha_h|$ is useful only as a stress comparison. A theorem for arbitrary coefficients would be stronger than the route needs; a theorem for absolute coefficients would erase the signs the route is attempting to exploit. The review should therefore ask for three columns in any M9 numerical or theorem-comparison table: actual coefficient, random phase stress, and absolute-value stress.

For Li--Yang, A1 and A3 correctly treat the paper as a guardrail rather than as a proof input. The relevant comparison is not merely that the phase resembles a reciprocal phase. The imported theorem would need to allow endpoint height
$$
H_D\asymp D X^{-1/4},
$$
the fixed Vaaler taper, residue restrictions in $h$, shifted phase variants for $\mathcal M_2$, and the appropriate amplitude class. Until those hypotheses are written in one theorem statement, the endpoint application remains unavailable. This is a source-hypothesis gap, not a numerical issue.

For H13, the useful content is narrower than a route verdict. A3's toy model says that after a specific B-process-first transform and a standard Gram-matrix norm extraction, the character can enter as a diagonal unitary conjugation. That diagnostic should be retained because it predicts sign erasure under a common proof move. ## Boundary-case audit

The first boundary case is small Vaaler height. At very small $X$ or at the lower dyadic endpoint, $H_D$ can be $0$ or $1$ depending on rounding. The proof route defines active blocks by $X^{1/4}\le D\le X^{1/2}$ and chooses $H_D\asymp D X^{-1/4}$. A numerical implementation should record the exact integer choice
$$
H_D=\max(1,\lfloor cD X^{-1/4}\rfloor)
$$
or another specified convention. Without this line, toy examples with $H_D=1$ cannot be compared across agents. This is why A3's $X=16,256$ examples are useful as regression checks but weak as asymptotic indicators.

The second boundary case is the nearest-integer tie in R5. If $X/d=m+1/2$, either nearest integer convention gives a residual displacement of size $1/2$. The product-count proof does not depend on the choice, but reproducible code does. A table should state whether ties are rounded up, rounded down, or counted twice with a harmless factor. The shifted legs have an analogous issue for $(X/d+\rho)/4$ near a half-integer. This can be handled with integer congruences rather than floating-point comparisons.

The third boundary case is complex dyadic weights. The paired real formulas use
$$
A_{-h}(D;X)=\overline{A_h(D;X)}
$$
and the analogous identity for $B_h$. This identity depends on real $w_D(d)$. If a Mellin transform, spectral decomposition, or smoothing procedure introduces complex weights, the raw two-sided definitions should be used. A theorem statement that silently imports the paired formulas into a complex-weight setting would change the object being estimated.

The fourth boundary case is parity in the $\mathcal M_2$ fourth moment. Since
$$
C_h=e(h/4)-e(3h/4)
$$
vanishes for even $h$, every exact-bin enumeration should exclude even $h_i$ before classifying families. If it classifies first and filters later, the family labels can be misleading. In particular, denominator-paired families can look large before parity restrictions and smaller after residue filtering. The review should ask A3 to report both pre-filter and post-filter counts when debugging the enumeration.

The fifth boundary case is the sign of the cleared integer
$$
N=h_1d_2d_3d_4-h_2d_1d_3d_4+h_3d_1d_2d_4-h_4d_1d_2d_3.
$$
Changing the order of conjugated factors changes signs in this expression. The review should require a printed convention line before any bin table. A table using $S_2^2\overline{S_2}^2$ and a table using $S_2\overline{S_2}S_2\overline{S_2}$ may be algebraically equivalent after relabeling, but their displayed $N$ formulas differ. Reproducibility depends on knowing which convention was used.
